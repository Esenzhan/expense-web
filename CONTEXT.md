# Контекст проекта — для Claude Code

Конспект для продолжения работы в новом чате. Проект спроектирован в claude.ai,
затем несколько больших сессий в Claude Code довели его до текущего состояния.

## Что это за проект

Веб-версия личного трекера трат (наследник telegram-expense-bot) — PWA для
iPhone («Добавить на экран Домой»), с мгновенным streaming-распознаванием
голоса. UI копирует референс-приложение «Qalta» (пользователь присылает
скриншоты/видео из него и просит повторять 1:1).

- Репозиторий: `github.com/Esenzhan/expense-web`
- Прод фронтенд: `https://expense-web-eosin.vercel.app` (Vercel, аккаунт
  `esenzhan`, scope `tommili1`, проект `expense-web`)
- Прод бэкенд: `https://expense-web-peo6.onrender.com` (Render free tier,
  автодеплой из GitHub `main`)
- БД: Neon Postgres (проект `expense-web`)

## Стек и архитектура

```
iPhone PWA (standalone)
  │ MediaRecorder webm/opus, чанки 250мс
  │ (клиент буферизует чанки до открытия WS + 700мс на рукопожатие
  │  сервер→Deepgram; первый чанк несёт заголовок контейнера!)
  ▼
WebSocket wss://…/ws/voice   ← при «стоп» клиент шлёт {"type":"stop"}
  ▼
Deepgram streaming STT (language=ru, nova-2, interim_results, endpointing=500)
  │ partial → на фронт live; сервер тоже буферизует аудио до открытия DG-сокета
  │ стоп-маркер → сервер шлёт Deepgram CloseStream → мгновенный финал
  ▼
Claude Haiku (claude-haiku-4-5-20251001) парсит текст → {amount, category,
  wallet, description}; списки категорий и кошельков подставляются в промпт
  ИЗ БД (кэш 60с, инвалидация при изменениях)
  ▼
Сервер шлёт {type:"parsed", proposal} — НЕ сохраняет сам!
  ▼
Фронт показывает карточку подтверждения (Отмена/Сохранить) → POST /api/expenses
```

- **Backend** (`backend/src`): Express + ws + pg + Anthropic SDK, один процесс.
  Таблицы: `expenses`, `categories` (seed 10 штук, «Прочее» = sort 999,
  фолбэк парсера, защищено от удаления), `wallets` (seed: Личные/Семья/
  Бизнес/Ремонт; «Личные» — фолбэк, защищён). Роуты: `/api/expenses`
  (GET/POST/PUT/DELETE), `/api/stats/{summary,insights,by-wallet,daily}`
  (summary и insights принимают `?wallet=`), `/api/categories` (GET/POST/
  DELETE), `/api/wallets` (GET/POST/PUT/DELETE; PUT переименовывает и
  переносит траты транзакцией; DELETE блокируется при наличии трат).
  `/api/stats/insights` — чистая арифметика без LLM (computeInsights.js,
  таймзона Asia/Almaty фиксированным смещением UTC+5).
- **Frontend** (`frontend/src`): React + Vite, без UI-библиотек, стили в одном
  `styles.css`. Recharts УДАЛЁН — единственный график (Инсайты) нарисован
  вручную SVG (`InsightsChart.jsx`). Реестры категорий/кошельков —
  модульные сторы (`categoryIcons.js`, `wallets.js`) с hydrate из API +
  localStorage-кэш.

## Экраны и фичи (всё уже на проде)

- **Главная**: чип кошелька (тап → шторка «Счета»), 4 круглые кнопки в шапке
  (поиск/статистика/кошельки — визуальные заглушки, шестерёнка работает),
  карточка «Расходы за» с переключателем периода (месяц/7/30) и кнопкой
  «✦ Инсайты», список трат по датам, док из 3 кнопок (скан-заглушка →
  открывает ручной ввод, микрофон 77px, «+»). SVG-иконки единого стиля.
- **Голос**: тап микрофона → live-транскрипт → карточка подтверждения.
  Отмена записи ✕. Fallback-таймер 15с.
- **Ручной ввод/редактирование** (`EditExpenseSheet`): калькулятор с
  +−×÷ (useReducer — НЕ useState, была потеря нажатий из-за батчинга),
  карусель категорий со snap к центру (выбор = что в центре), выбор
  кошелька, заметка, удаление за два тапа в меню ⋮. Тап по строке траты
  открывает редактор (раньше сразу удалял!).
- **Инсайты** (`InsightsSheet`): открывается тапом ИЛИ тягой вниз из любого
  места главной (только при scrollY=0; кнопка растягивается по мере тяги).
  Почти во весь экран (position:absolute, top:7dvh, bottom:0 — НЕ height:vh,
  на iOS глючило). График: факт/проекция/прошлый период; 6 карточек метрик.
  Фильтруется по выбранному кошельку.
- **Настройки** (`SettingsSheet`, страница въезжает справа): все секции из
  референса; работают: Категории, Экспорт CSV (реальный), 5 переключателей
  (persisted в localStorage). Остальное — визуальные строки. Pro-баннер и
  чужие соцсети НЕ копировали.
- **Категории** (`CategoriesSheet` + `NewCategorySheet`): список с ручками
  (перетаскивание НЕ реализовано, только вид), создание своей категории:
  имя + live-превью, палитра 12 цветов, сетка emoji по группам,
  Suggestions по ключевым словам («Дом и быт» → 🏠🛋️🛏️…).
- **Счета** (`WalletsSheet` + `NewWalletSheet`): «Все счета» + список с
  суммами месяца, выбор фильтрует ВСЮ главную (чип/сумма/список/Инсайты,
  persisted), карандаш → редактирование, «+» → создание (те же
  палитра/иконки/Suggestions). Новые счета/категории сразу доступны голосу.
- **Шторки** (`sheetGestures.js`): все закрываются свайпом вниз (только когда
  внутренний скролл наверху; горизонтальные тяги не перехватываются —
  карусель), затемнение гаснет пропорционально тяге, размонтирование по
  transitionend. Скролл фона глушится НА УРОВНЕ КАСАНИЙ (preventDefault на
  затемнении + overscroll-behavior) — position:fixed на body ломает iOS
  (шторка всплывает над низом), overflow:hidden сбрасывает скролл.

## Захардкоженные знания о платформе (не переоткрывать!)

- **Вибрация на iOS Safari**: Vibration API нет. Хак — программный клик по
  label скрытого `<input type="checkbox" switch>` (`haptics.js`). Работает
  iOS 17.4–26.4, ТОЛЬКО из обработчиков настоящего click (синхронно).
  Из touch-обработчиков (start/move/end) НЕ работает никак — вибрации при
  свайпе на iPhone в вебе не бывает, это потолок платформы. setTimeout
  убивает даже рабочие случаи. На Android — navigator.vibrate везде.
- **iOS PWA восстанавливается из памяти** без перезагрузки — пользователь
  может видеть старую версию, пока не убьёт приложение из переключателя.
- **Deepgram**: финализирует фразу только по ≥500мс тишины В ПОТОКЕ или по
  CloseStream. Обрыв аудио без стоп-маркера = вечное «Обрабатываю…».
  Аудио, пришедшее до открытия сокета (клиентского или серверного к DG),
  раньше молча ДРОПАЛОСЬ — терялся заголовок контейнера и весь поток
  становился непарсибельным. Теперь буферизуется с обеих сторон.
- **Service worker** (`public/sw.js`): HTML — network-first (иначе деплой
  залипает на старой версии), хэшированные ассеты — stale-while-revalidate.
- **Vercel**: аккаунт залогинен в CLI (`esenzhan`). При сбоях GitHub деплоить
  напрямую: `npx vercel deploy --prod --yes` из корня репо (проект слинкован,
  `.vercel/` в gitignore). Частый curl на прод-домен ловит Vercel Security
  Checkpoint (антибот) — проверять через браузер.
- **Render free tier**: холодный старт 30–50с — все таймауты/буферы это
  учитывают.

## Как тестировать без телефона

- Синтез русской речи: `say -v Milena -o x.aiff "текст"` + `afconvert -f WAVE
  -d LEI16@16000 -c 1`. Прогон через прод-WS из Node напрямую НЕ
  воспроизводит клиента честно (WAV с фиксированной длиной ≠ живой поток).
  Честный тест: страница в браузер-пейне, WAV → WebAudio →
  MediaStreamDestination → MediaRecorder(webm/opus) → прод-WS (см. историю:
  scratchpad/browser_voice_test.js; скрипт эфемерный, при надобности
  пересоздать).
- Жесты в браузер-пейне: синтетические TouchEvent/Touch диспатчем на элемент.
- Кадры из видео-референсов: swift-скрипт с AVAssetImageGenerator (эфемерный
  /tmp/extract_frames2.swift — пересоздать при надобности).
- В браузер-пейне таймеры/анимации троттлятся — замеры «через N мс» врут,
  доверять итоговому состоянию DOM.

## Открытые хвосты

1. **Не подтверждено пользователем**: высота Инсайтов на телефоне после фикса
   с absolute-позиционированием (коммит 4affbfe). Если всё ещё полэкрана —
   копать рендер конкретной iOS.
2. Сканирование чека — кнопка есть, открывает ручной ввод. Полноценное
   распознавание (фото → Claude Vision → карточка) не делалось.
3. Кнопки шапки поиск/статистика/кошельки(cylinder) — декоративные.
4. Перетаскивание категорий (ручки ⠿) — только визуал, reorder не реализован.
5. Доход/Перевод — везде только «Расход» (тумблеры в UI визуальные).
6. Старый TODO из бота: детерминированный keyword-матчинг кошелька ДО Haiku
   (файл с правилами бота так и не был предоставлен).
7. Удаление категорий/кошельков есть в API, но НЕ в UI.

## Стиль работы с пользователем

Пользователь (Дария) присылает скриншоты/видео из Qalta и просит копировать
точь-в-точь; тестирует на iPhone и отвечает коротко («опять завис», «сделай
так же»). Прод-первый цикл: коммит → push (Render сам) + `npx vercel deploy
--prod --yes` → проверка на проде через браузер-пейн → просьба проверить на
телефоне. Коммиты — содержательные, на английском, без Co-Authored-By (у
пользователя git identity не настроен — предлагали, не просил).
