# Траты — голосовой веб-апп

Веб-версия telegram-expense-bot: те же 4 кошелька (Личные / Семья / Бизнес / Ремонт)
и парсинг через Claude Haiku, но с графиками и **живым (streaming)** распознаванием
речи вместо записи-целиком-потом-Whisper.

## Как это работает

```
iPhone (PWA, добавлен на экран Домой)
  │  MediaRecorder режет звук на чанки по 250мс
  ▼
WebSocket  wss://backend/ws/voice
  │  каждый чанк сразу пробрасывается дальше
  ▼
Deepgram (streaming STT, language=ru)
  │  партиал-текст летит обратно на фронт в реальном времени (эффект "мгновенно")
  │  финальный текст (после ~500мс тишины) идёт дальше
  ▼
Claude Haiku — парсит финальный текст в {amount, category, wallet, description}
  ▼
Postgres (Neon) — сохраняет трату
  ▼
Фронт получает {type: "saved", expense} и обновляет списки/графики
```

## Backend (`/backend`)

- Express + `ws` (WebSocket) на одном HTTP-сервере
- `POST/GET/DELETE /api/expenses`, `GET /api/stats/*` — обычный REST для списка и графиков
- `wss://.../ws/voice` — стрим аудио → партиал/финальный текст → сохранённая трата
- Postgres: одна таблица `expenses` (schema создаётся автоматически при старте)

Переменные окружения — см. `backend/.env.example`:
- `DATABASE_URL` — как у [[saubol]] / [[kaspi-dashboard]], можно новый проект в Neon
- `ANTHROPIC_API_KEY` — как в telegram-expense-bot, но лучше отдельный ключ в консоли
- `DEEPGRAM_API_KEY` — регистрация на deepgram.com, есть бесплатный tier
- `FRONTEND_URL` — для CORS

Деплой — как обычно у вас: Render (Root Directory `backend`, Build `npm install`,
Start `npm start`). **Важно:** Render должен поддерживать WebSocket на выбранном
плане — на free-инстансах это работает, но проверьте, что прокси не режет
long-lived соединения.

## Frontend (`/frontend`)

- React (Vite) + Recharts, без Tailwind — стили в `src/styles.css`
  (тема "гроссбух/чек": тёмный чернильный фон, золотой акцент, суммы моноширинным шрифтом)
- `manifest.json` + мета-теги `apple-mobile-web-app-*` в `index.html` —
  чтобы после «Добавить на экран Домой» приложение открывалось как отдельное
  приложение, без адресной строки Safari
- `VITE_API_URL` — адрес бэкенда (см. `frontend/.env.example`)

Деплой на Vercel (Root Directory `frontend`, Vite preset) — `vercel.json` уже
содержит rewrite для SPA-роутинга.

## Важные нюансы iOS

1. **Не полагаемся на Web Speech API** (`webkitSpeechRecognition`) — на iOS Safari,
   особенно в установленном PWA, он нестабилен (глючит `continuous`,
   `interimResults` иногда молча переключается на медленное облачное
   распознавание). Вместо этого — свой стрим через `MediaRecorder` + Deepgram,
   который не зависит от этих багов.
2. **Фоновая работа ограничена.** Если свернуть PWA во время записи, iOS может
   прервать сеть/микрофон. UX рассчитан на «держим экран открытым, пока
   говорим» — как у Qalta на скриншоте.
3. **`MediaRecorder` mimeType** — Safari отдаёт `audio/mp4`, Chrome/Android —
   `audio/webm;codecs=opus`. Код уже выбирает поддерживаемый тип автоматически.

## Мгновенное открытие (как в Telegram-боте)

Три вещи вместе дают эффект «открыл и сразу пишу/говорю», без спиннера:

1. **Service worker (`public/sw.js`)** кеширует HTML/JS/CSS/иконки
   (stale-while-revalidate). После первого открытия все статические файлы
   отдаются из кеша устройства мгновенно, а обновление подтягивается в
   фоне — так же, как обновляется сам Telegram между запусками.
2. **Кеш последних данных в `localStorage`** (`App.jsx`) — при открытии
   экран рисуется сразу с теми тратами/графиками, что видели в прошлый раз,
   а свежие данные с бэкенда подменяют их в фоне без перерисовки в пустоту.
3. **Ленивая загрузка графиков** (`React.lazy`) — самая тяжёлая часть
   бандла (Recharts) подгружается уже после того, как кнопка микрофона и
   список трат стали интерактивны, так что записывать голосовое можно,
   не дожидаясь графиков.

Голосовая кнопка и раньше не ждала данных — она всегда была отдельным,
всегда смонтированным компонентом внизу экрана, так что с этими правками
диктовку можно начинать буквально в момент открытия приложения.

## Локальный запуск

```bash
cd backend && npm install && cp .env.example .env  # заполнить ключи
npm run dev

cd frontend && npm install && cp .env.example .env
npm run dev
```

Открыть `http://localhost:5173` с телефона в одной Wi-Fi сети (или задеплоить
и открыть по HTTPS — микрофон в браузере требует either localhost, либо HTTPS).
